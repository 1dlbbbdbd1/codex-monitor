# Codex Floating Companion 设计规格

日期：2026-07-12

状态：已获用户批准，可进入实施计划与开发

## 1. 目标

在 Windows 上提供一个轻量的 Codex 桌面伴侣：持续显示 Codex 5 小时和 7 天额度及刷新时间，并通过可拖动、可贴边收起的悬浮球汇总全部 Codex 任务的运行、审批、完成和失败状态。

第一版以 MIT 许可的 `ademisler/codexcontrol` Windows 实现为基座，复用其额度读取、认证、托盘、开机启动、账户存储、测试和 PyInstaller 发布流程。新增能力通过边界清晰的模块实现，避免继续扩大原项目的 `app.py`。

## 2. 产品范围

### 2.1 第一版必须具备

- 显示当前账户的 5 小时和 7 天额度剩余百分比。
- 显示每个额度窗口的准确刷新时间和数据更新时间。
- 每 5 分钟自动刷新额度，并允许手动刷新。
- 聚合全部已观测 Codex 任务，显示空闲、工作中、等待审批、完成、失败和未知状态。
- 以约 56 像素的置顶悬浮球显示聚合状态和工作中任务数量。
- 支持拖动、左右边缘吸附、多显示器定位和位置持久化。
- 吸附后闲置约 2 秒自动缩入边缘，仅保留可点击把手。
- 等待审批时显示持续红点，直至审批完成。
- 任务完成、失败或额度刷新时显示红点，用户打开面板查看后清除。
- 工作中仅显示动画和任务数量，不显示红点。
- 点击悬浮球展开面板，显示额度和任务列表。
- 点击任务时优先打开对应 Codex 任务；若当前版本不支持精确跳转，则激活 Codex 窗口并显示项目名和任务标题。
- 状态链路失效时明确显示“状态未知”，额度链路继续工作。
- 安装器支持开机启动，并安全安装/更新状态桥，不覆盖用户已有 hooks。
- 生成可运行的 Windows 单文件 Demo。

### 2.2 第一版不包含

- 自动批准或拒绝 Codex 请求。
- 远程控制、手机推送或云同步。
- Claude、Cursor 等其他 AI 工具的状态采集。
- 主题商店、宠物系统或复杂皮肤编辑器。
- 修改 Codex 对话正文或上传对话内容。

## 3. 架构

系统由五个独立单元组成：

1. `QuotaService`：沿用 CodexControl 的认证与额度读取逻辑，产生标准化额度快照。
2. `ActivityBridge`：由 Codex hooks 调用的轻量命令行入口，将最小化事件原子追加到本地 JSONL 日志。
3. `ActivityStore`：增量读取事件，去重、恢复任务状态、执行超时降级并持久化确认状态。
4. `StatusAggregator`：把所有任务和额度事件归并为悬浮球展示模型。
5. `FloatingOverlay`：负责悬浮球、贴边把手、红点、展开面板和跳转动作。

现有账户管理和主窗口保持可用。新模块只通过数据模型和回调与主应用通信，不直接操作认证文件。

## 4. 本地数据与接口

### 4.1 事件格式

状态桥每次写入一行 JSON，字段如下：

```json
{
  "schemaVersion": 1,
  "eventId": "uuid",
  "eventType": "turn_started",
  "threadId": "thread-id",
  "turnId": "turn-id-or-null",
  "taskTitle": "optional short title",
  "projectName": "optional project name",
  "cwd": "optional local path",
  "occurredAt": "2026-07-12T12:00:00Z",
  "source": "hook"
}
```

允许的 `eventType`：

- `turn_started`
- `approval_requested`
- `approval_resolved`
- `turn_completed`
- `turn_failed`
- `turn_interrupted`
- `heartbeat`

事件不得包含提示词、回复正文、命令正文、diff、环境变量或认证信息。

### 4.2 存储位置

应用数据统一位于 `%APPDATA%\CodexFloatingCompanion`：

- `activity-events.jsonl`：状态桥追加事件。
- `activity-state.json`：任务投影、最后读取偏移和已确认提醒。
- `settings.json`：悬浮位置、吸附边、自动收起和启动设置。
- `logs/app.log`：轮转诊断日志，默认不记录任务标题和路径。

额度和账户数据继续使用 CodexControl 原有存储，但迁移到新产品目录时必须兼容读取旧路径。

### 4.3 hooks 安装

安装器读取用户现有 `%USERPROFILE%\.codex\hooks.json`：

- 不存在时创建合法配置。
- 存在时解析 JSON 并只添加带稳定标识 `codex-floating-companion` 的命令 hook。
- 已存在同标识配置时原位升级，不重复追加。
- 写入前生成带时间戳备份，使用临时文件加原子替换。
- JSON 无法解析时停止安装状态桥，保留额度功能并向用户显示修复说明。
- 卸载时只移除本产品标识的 hook，不触碰其他条目。

优先映射 `UserPromptSubmit`/回合开始、`PermissionRequest`、`Stop`/完成等当前可用事件。具体 Codex 版本缺少某事件时，状态显示为降级而不是猜测。

## 5. 状态模型

单任务状态为：`unknown`、`idle`、`working`、`waitingApproval`、`completed`、`failed`。

聚合优先级固定为：

1. `waitingApproval`
2. `failed`
3. `working`
4. `completed`
5. `idle`
6. `unknown`

当多个任务同时工作时，悬浮球显示工作中数量。等待审批时显示待审批数量并持续红点。

任务在 `turn_started` 后进入 `working`；`approval_requested` 进入 `waitingApproval`；`approval_resolved` 恢复到 `working`；完成、失败或中断事件进入相应终态。未收到终态且事件长时间无更新时，不直接判定完成：若 Codex 进程和 session 日志都无法证明仍在活动，则转为 `unknown`。

## 6. 提醒状态机

- 审批提醒：事件到达即设为未处理；只有 `approval_resolved` 或对应任务终态才能清除。
- 完成/失败提醒：事件到达即设为未读；用户打开展开面板后标记已读。
- 额度刷新提醒：当窗口此前剩余不足 100%，且 `resetAt` 跨过当前时间后新的快照恢复到更高额度，设为未读；首次启动不触发。
- 工作中状态不产生红点。
- 面板打开不会清除仍未解决的审批红点。

## 7. 悬浮界面

### 7.1 悬浮球

- 默认直径 56 像素，支持 100%、125%、150%、175% 和 200% DPI。
- 无任务时静态显示 Codex 标识；工作中使用低频旋转/呼吸动画；等待审批显示红色徽标；失败显示红色边框；状态未知显示灰色断连标识。
- 数字只表示工作中或待审批任务数量，超过 9 显示 `9+`。
- 拖动结束后保存所在显示器、归一化坐标和吸附边。

### 7.2 贴边与多显示器

- 距离当前显示器工作区左右边缘 24 像素内释放时吸附。
- 吸附后 2 秒无鼠标悬停自动收起，保留约 12 像素把手。
- 红点完整位于可见工作区内。
- 鼠标进入把手或点击时滑出；展开面板期间不自动收起。
- 显示器拔出或分辨率变化后，将窗口恢复到最近的可用工作区。

### 7.3 展开面板

面板紧邻悬浮球并自动避开屏幕边缘，包含：

- 当前账户、5 小时与 7 天额度、刷新时间、最后更新时间和手动刷新按钮。
- 状态连接标识：正常、降级、未知。
- 任务列表：状态、项目名、任务标题、最近更新时间。
- 托盘/设置入口：开机启动、悬浮球开关、自动收起开关、重新安装状态桥、退出。

## 8. Codex 跳转

第一优先级是使用当前 Codex 桌面客户端支持的任务导航能力。若没有公开或稳定的精确导航接口：

1. 激活 Codex 主窗口。
2. 将目标任务在伴侣面板中保持高亮。
3. 显示项目名和任务标题，不使用自动键鼠操作猜测 UI 位置。

不得修改 Codex 打包文件、SQLite 状态或私有 IPC。

## 9. 故障处理

- 额度请求失败：保留最后成功快照并显示陈旧时间；不得将缓存伪装成实时数据。
- hooks 配置失败：额度功能继续，状态连接显示“未安装”或“配置错误”。
- JSONL 尾部出现半行：保留偏移，下次继续读取；坏行隔离到诊断日志，不中断应用。
- 重复事件：按 `eventId` 去重；缺失 ID 的兼容事件按线程、类型、回合和时间桶去重。
- 应用崩溃重启：从快照和事件偏移恢复，不重复弹红点。
- 状态源冲突：hook 事件优先于日志推断；冲突无法消解时标记 `unknown`。

## 10. 安全与隐私

- 仅使用 Codex 已有的本地认证状态获取额度，不要求 OpenAI API Key。
- 不上传遥测、不运行本产品后端、不记录会话正文。
- 状态桥只接受固定事件类型和有限字段；所有文本字段有长度限制并进行控制字符过滤。
- hooks 合并、设置文件和状态快照使用原子写入。
- 发布包保留 CodexControl 原版权和 MIT 许可证，并明确为非 OpenAI 官方产品。

## 11. 测试与验收

### 11.1 自动测试

- 状态转换、聚合优先级和多任务计数。
- 审批、完成、失败和额度刷新红点规则。
- JSONL 增量读取、半行恢复、坏行隔离和事件去重。
- hooks 新装、合并、升级、损坏配置拒绝、备份和卸载。
- 悬浮位置计算、左右吸附、收起把手、多显示器恢复和 DPI 缩放纯逻辑。
- 额度服务现有测试全部保持通过。
- PyInstaller 打包烟雾测试。

### 11.2 Windows 实机验收

- Codex 空闲、工作、请求审批、审批完成、任务完成和失败时状态正确。
- 同时运行多个任务时数量和优先级正确。
- 左右贴边、自动收起、红点常显、悬停滑出和位置恢复正确。
- 100%、125%、150% DPI 以及双显示器切换不丢失窗口。
- Codex 未运行、hooks 不可用和网络断开时降级文案准确。
- 安装、升级、开机启动和卸载不会损坏用户已有 hooks。
