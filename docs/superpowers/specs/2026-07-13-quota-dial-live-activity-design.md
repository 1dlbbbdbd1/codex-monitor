# 额度悬浮球与实时任务修复设计

## 目标

- 悬浮球中央直接显示剩余额度百分比。
- 用户可在 `5 小时` 和 `7 天` 额度之间切换，并始终看得出当前口径。
- Codex 实时任务事件能由受支持的同步 hook 写入，不再被 Codex 跳过。
- 继续使用现有 Tk、Pillow、JSON 设置和 JSONL 活动管线，不增加依赖。

## 交互

- 默认显示 `5 小时`；中央为大号百分比，下面显示小号 `5小时` 或 `7天`。
- 外圈按剩余百分比绘制，充足为绿色、偏低为黄色、紧张为红色。
- 展开面板的额度标题旁提供 `5小时 / 7天` 两段切换按钮，选择持久化。
- 如果服务端没有当前所选窗口，自动显示实际存在的另一个窗口；不可用的切换项不执行切换。
- 任务状态保留在悬浮球右上角状态点和面板标题中，不再占用中央百分比区域。

## 实时任务数据流

保留 `Codex hooks -> bridge_cli -> activity-events.jsonl -> ActivityStore -> OverlayViewModel`。安装器和插件模板删除 `async: true`，因为 Codex 当前会跳过异步命令 hook。修复入口重新写入四个同步 hook；更新后的 hook 仍需用户在 Codex `/hooks` 中确认信任。

## 验证

- 设置往返保存额度口径。
- 视图模型覆盖默认选择、手动切换和缺失窗口回退。
- 图标测试覆盖进度环随百分比变化且保持抗锯齿。
- hook 安装测试断言四个处理器都不包含 `async`。
- GUI 冒烟截图确认百分比、口径标签、状态点和面板无重叠。
